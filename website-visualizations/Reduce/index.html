🔻 Reduce Algorithm Visualizations

Interactive visualizations of distributed reduction algorithms on 2D and 3D mesh networks

What is Reduce?

Reduce is a collective operation where all processes contribute data that gets combined (e.g., summed) at a single root process. This is the reverse of broadcast - data flows from many sources to one destination.

In a mesh network, each node starts with a local value (typically 1), and these values are aggregated at the root node (0,0) using different routing strategies. The goal is to minimize communication time and path conflicts.

Each algorithm below represents a different approach to performing this reduction operation efficiently on 2D and 3D mesh topologies.

🔴

Brute Force

Sequential reduction where the root (0,0) receives messages from all N²−1 nodes one at a time. Each source routes its data hop-by-hop to the root.

Complexity: O(N³) for 2D, O(N⁴) for 3D
Simple but inefficient baseline approach
🌲

Inclusion-Exclusion

Wavefront reduction using inclusion-exclusion dependencies. Nodes combine values from neighbors and propagate results diagonally toward the root.

Complexity: O(N²) for 2D, O(N³) for 3D
Dependency-driven diagonal wave propagation
📊

Row-Column Pipeline

Two-phase reduction: first reduce within rows (east→west), then reduce column leaders vertically (south→north) to the root.

Complexity: O(N) for 2D, O(N) for 3D
Efficient pipelined approach with parallel reduction
🔄

Recursive Quadrant

Recursively divide the grid into quadrants (2D) or octants (3D). Each region reduces internally, then sends its result to the parent region's root.

Complexity: O(N) for 2D, O(N) for 3D
Divide-and-conquer with recursive subdivision
🔲

Hierarchical (Block-Based)

Divide into √N blocks of size √N×√N. Each block reduces internally using row-column pipeline, then block leaders send to the global root.

Complexity: O(√N) for 2D, O(N^(2/3)) for 3D
Optimal two-level hierarchical reduction
